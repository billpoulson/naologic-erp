# Architecture Decision Records (ADRs)

This folder contains Architecture Decision Records for key technical and design decisions.

## Format

Each ADR should follow this structure:

- **Title**: `ADR-XXXX: [Short descriptive title]`
- **Status**: Proposed | Accepted | Deprecated | Superseded by [ADR-YYYY]
- **Context**: What is the issue or situation?
- **Decision**: What was decided?
- **Consequences**: What are the implications?

## Naming

Use sequential numbers with a short slug: `0001-short-description.md`
